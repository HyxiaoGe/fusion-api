# Fusion 生产全链路压测闭环计划

## 目标

在唯一生产环境补齐 L1-L4 性能矩阵，保留可复跑、默认脱敏、带硬停止和精确清理的 runner，并把最终安全摘要导入管理员审计中心。

## 环境与门禁

- 目标：`https://fusion.seanfield.org`，必须显式确认生产。
- API 当前限制：1 vCPU、1 GiB、单 Uvicorn worker。
- HTTP：至少 20 样本后错误率或超时率达到 5%，或连续失败达到 10 次，停止升压。
- SSE：任一 flow 失败、error frame、清理失败，停止升压。
- 主机/API/PostgreSQL/Redis 出现 OOM、容器重启、Redis rejected connection/eviction 增量，立即停止。
- 所有密码、access token、refresh token、邮箱和响应正文只存在进程内存，不写结果。

## 用例矩阵

| 层级 | 场景 | 负载 | 核心指标 | 通过条件 |
|---|---|---:|---|---|
| L1 | 登录 | c10→25→50 | RPS、P50/P95/P99、错误率、DB 连接 | 0 超时；错误率 <5%；无资源硬门禁 |
| L1 | 模型列表 | c10→25→50 | 同上 | 同上 |
| L1 | 会话列表 | c10→25→50 | 同上 | 同上 |
| L1 | 会话详情 | c10→25→50 | 同上 | 同上 |
| L2 | 短流式对话 | c1→3→5→10 | TTFT、完成耗时、chunk 间隔、输出速率、完成率 | 100% 完成；0 error frame |
| L2 | 长流式对话 | c1→3→5 | 同上 | 100% 完成；0 error frame |
| L3 | 状态恢复/断线重连 | c1→3→5 | 恢复成功率、重复/丢失、恢复耗时 | 100% 恢复；0 重复/丢失 |
| L3 | 停止生成 | c1→3→5 | 停止成功率、停止延迟、终态/残留 | 100% 停止；无活跃任务残留 |
| L4 | 低并发稳态 | c2，30 分钟 | 窗口错误率、TTFT、内存斜率、连接/Swap、错误累积 | 无错误累积、重启、OOM、连接泄漏 |

## 实施顺序

1. 先补 runner 的场景、指标和纯函数测试，不启动本地 Fusion 服务。
2. 生产运行前记录数据库计数、容器状态和 Prometheus 基线。
3. 逐层运行；每层结束检查应用日志与资源指标，满足门禁才继续。
4. `finally` 精确删除压测会话并吊销全部 refresh token；再按 `run_id` 精确清理 Fusion/auth 一次性用户和登录日志。
5. 生成脱敏 JSON 与 Markdown 报告，导入管理员审计中心，在已打开且已登录的 Fusion Chrome 标签验证持久化和页面可见性。
6. 更新执行台账，运行测试、Ruff、架构检查，提交、push 并跟踪 CI/CD。
